# START HERE — Review PR

> **Note:** This exercise is done on GitHub's web interface. No terminal commands needed.

1. [ ] Open an existing PR in this repo (or ask a partner to open one)
2. [ ] Read the diff and description; check against CONTRIBUTING.md (branch name, commit messages)
3. [ ] Leave at least one comment (e.g. on a line or in the main thread) — suggestion or question
4. [ ] If everything looks good, approve; otherwise "Request changes" with clear feedback
5. [ ] Fill or tick the review checklist if the repo has one

Check [expected-result.md](expected-result.md).
